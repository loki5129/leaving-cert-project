Investigation

My first idea was to make a project using the different types of energy used for electricity in Ireland. My second idea was to use a Pokémon dataset to find the base stat total and find which type had the highest mean value. My third idea, which is the one I decided to use, was to use a dataset containing Pokémon trainers and their Pokémon to find out the most common types and average level of Pokémon. I found this dataset on Kaggle [1]. I started by researching Pokémon levels using search engines such as Google and DuckDuckGo. From an article on Bulbapedia[2]I found out that Pokémon levels have a range of 1 to 100. I then researched similar projects and I identifed a similar project on Reddit[3], although this project was similar it only included one game and the frequency of Pokémon used by trainers. I researched Pokémon types, I learned that there are 20 types in the Pokémon games[4] 2 of these types are not in the dataset because they are only used in the latest game, and the dataset was created 8 years ago. I researched what data was in the dataset. The games that are included in the dataset are shown in Figure.1. The games highlighed in green are in the dataset and those in red are not in the dataset .To make this table I found a unique trainer in the game using a walkthrough on Bulbapedia[5] and cross checked it to see if it was in the dataset.This dataset also includes rematches that can be done after defeating a trainer the first time[6]. I also discovered that it does not include gym leaders[7],rivals[8],elite four[9] and champions[10].

Problem Encountered

One of the problems that I encountered when making this project was getting the graph from the Python code to the JavaScprit code.I first attempted to fix this problem using Flask after research I decided not to use it has it was new to me and seemed more complex than I needed for this problem. The second idea was to use the bulit-in function in Plotly to export an intreactive graph, after looking at the documation for this function I learned that you can use Jinja2 to place it inside a HTML file in a specific place. This worked to get the graph onto the webpage but when trying to view the HTML code for the webpage it crashed the IDE. To Fix this I researched and found that adding conditions to the function can reduce its size. Adding conditions stopped the function including the entirety of the Plotlyjs and Mathjax libraries in the output and instead I linked them as script files in the header. I set full_html to false so that it only sends the necessary code. When I reviewed the output it was a div with an unique id and the graph inside a scprit tag. I had the idea to put the graph code that is inside the scprit tag into a JavaScprit file and the div with the id into a HTML file and link the JavaScprit to the HTML. To do this I used the String method to get the section of the output that contains the div with the id and place it in a HTML file using the Jinja2 template function. Then using the same String method I took the section that contains the graph and removed the parts that are incompatible with JavaScprit and placed the cleaned code into a JavaScprit file using the same Jinja2 function. This made the graph show cleanly and it can be easily viewed and debugged.
